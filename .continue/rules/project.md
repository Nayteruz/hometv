# Архитектура проекта "Домашний кинотеатр"

## Общее описание

Проект представляет собой веб-приложение для просмотра фильмов и сериалов с использованием API Кинопоиска и сторонних плееров. Пользователи могут просматривать фильмы, добавлять их в избранное и создавать персональные списки.

## Стек технологий

- **Frontend**: Vue 3, Composition API, Pinia (для управления состоянием)
- **Аутентификация**: Firebase с Google Auth
- **API**: Кинопоиск API (неофициальное), сторонние плееры
- **Структура**: TypeScript, SCSS, Vite

## Основные модули

### 1. Хранилища (Stores)

- `filmStore.ts` - основное хранилище для управления фильмами, фильтрами и поиском
- `authStore.ts` - управление авторизацией пользователя
- `userListsStore.ts` - пользовательские списки (избранное, последние просмотры, списки ожидания и т.д.)

### 2. Компоненты

- `FavoriteActionButton.vue` - кнопка добавления/удаления из избранного на странице списка фильмов
- `ActionFavorite.vue` - кнопка добавления/удаления из избранного на странице фильма
- `FilmPage` - компоненты для отображения детальной информации о фильме
- `Search` - компоненты поиска фильмов
- `Filters` - фильтрация по жанрам

### 3. API

- `src/components/api.ts` - модуль для работы с API Кинопоиска
- `src/components/utils.ts` - вспомогательные функции для работы с данными

### 4. Конфигурация

- `src/plugins/firebaseConfig.ts` - конфигурация Firebase
- `.env` файлы для хранения API ключей

## Важные соглашения

### 1. Структура файлов

- Все компоненты находятся в `src/components/`
- Хранилища в `src/stores/`
- Плагины в `src/plugins/`
- Типы данных в `src/types/`

### 2. Работа с API

- Все запросы к API осуществляются через `fetch` с заголовками
- Используются специальные ключи API из .env файлов
- Обработка ошибок в API-модуле

### 3. Управление состоянием

- Используется Pinia для управления состоянием
- Хранилища имеют строгую структуру с типами
- Состояние синхронизируется с Firebase при авторизации

### 4. Пользовательские списки

- Все пользовательские списки (избранное, последние просмотры и т.д.) хранятся в Firebase
- При добавлении/удалении элементов из списков происходит автоматическая синхронизация с сервером
- Списки имеют ограничения по количеству элементов

### 5. Аутентификация

- Авторизация через Google через Firebase
- Данные пользователя хранятся в Firebase
- Пользовательские списки привязаны к конкретному пользователю

## Основные особенности

- Поддержка авторизации через Google
- Хранение пользовательских списков в облаке
- Адаптивный дизайн
- Использование неофициального API Кинопоиска
- Интеграция с внешними плеерами
